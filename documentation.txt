Jira Clone Progress Guide (Step-by-Step)

This document records the exact steps completed so far and where the project currently stops.

Step 1: Initialize the project
- Created a Next.js 16 App Router project with TypeScript.

Step 2: Set up UI foundation
- Configured Tailwind CSS.
- Configured shadcn/ui.
- Generated base UI components used by auth pages.

Step 3: Install core dependencies
- Installed framework and UI dependencies:
	- next, react, react-dom
	- class-variance-authority, clsx, tailwind-merge
	- lucide-react, react-icons
	- sonner
- Installed form and validation dependencies:
	- react-hook-form
	- zod
	- @hookform/resolvers
- Installed API utility dependency:
	- hono
	-tanstack

Step 4: Create auth app structure
- Added grouped auth routes under app/(auth).
- Created pages for:
	- sign-in
	- sign-up

Step 5: Build sign-in form (client-side)
- Implemented React Hook Form using useForm.
- Used Controller to integrate custom Input components.
- Added Zod schema validation for:
	- email format
	- password length constraints
- Rendered field-level validation errors.

Step 6: Build sign-up form (client-side)
- Implemented React Hook Form using useForm.
- Used Controller for custom inputs.
- Added Zod schema validation for:
	- email
	- password
	- confirmPassword
- Added password confirmation check with refine.
- Attached mismatch error to confirmPassword using path ["confirmPassword"].

Step 7: Add field primitives and accessibility
- Used shadcn Field primitives:
	- Field
	- FieldGroup
	- FieldError
- Displayed per-field errors using FieldError + fieldState.error.
- Added aria-invalid to invalid inputs.
- Added noValidate on forms to rely on schema-driven validation messages.

Step 8: Current submit behavior
- Form onSubmit handlers currently log form values.
- No API call is wired yet.

Current Project Status (Reached Point)
- Sign-in: client-side validation is complete.
- Sign-up: client-side validation is complete.
- Confirm password mismatch handling is complete.
- Field-level error rendering is complete.
- Backend submission flow is not implemented yet.

2. Update sign-up submit handler to async API submission.
3. Map server field errors to form.setError.
4. Add generic top-level error handling for unknown failures.
