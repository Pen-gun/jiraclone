Jira Clone Technical Documentation

Version: March 2026

Purpose
- This document is a from-scratch onboarding runbook for new or junior developers.
- If you follow all steps in order, you should reach the current working state of this repository.

Current Product Scope
- Auth: sign up, sign in, onboarding, current-user, logout.
- Dashboard root page with server auth gate.
- Workspace create form and API.
- Note: workspace UI currently stores records in Prisma `Project` model.

Tech Stack
- Next.js 16 App Router
- React 19 + TypeScript
- Hono API mounted in Next route handler
- Prisma + PostgreSQL
- TanStack React Query
- React Hook Form + Zod
- Tailwind CSS + shadcn UI style components

Prerequisites
- Node.js 20 or newer
- npm 10 or newer
- PostgreSQL database available locally or remote
- Git installed

Onboarding Setup (From Scratch)

1. Clone and install
- `git clone <repo-url>`
- `cd jiraclone`
- `npm install`

2. Create environment files
- Create `.env` and `.env.local` in project root.
- Add values like:

`DATABASE_URL="postgresql://USER:PASSWORD@HOST:5432/DB_NAME?schema=public"`
`NEXT_PUBLIC_API_URL="http://localhost:3000"`
`NODE_ENV="development"`

3. Generate Prisma client and apply schema
- `npx prisma generate`
- `npx prisma migrate dev`

4. Start app
- `npm run dev`
- Open `http://localhost:3000`

5. First-run verification
- Open `/sign-up` and create an account.
- Complete onboarding.
- Confirm `/` loads and does not redirect to sign-in while logged in.
- Click logout and verify redirect to `/sign-in`.

Expected API Endpoints
- `POST /api/auth/register`
- `POST /api/auth/login`
- `POST /api/auth/onboarding`
- `POST /api/auth/logout`
- `GET /api/auth/me`
- `POST /api/workspaces`

- `app/api/[...route]/route.ts` mounts Hono routes.
- `features/auth/server/route.ts` auth endpoints.
- `lib/session-middelware.ts` cookie/session validation.
- `features/action.ts` server-side current-user gate for pages.
- `features/auth/api/*` auth client hooks.
- `features/workspaces/components/create-workspace-form.tsx` workspace create UI.
- `features/workspaces/server/route.ts` workspace create API.
- `features/workspaces/schemas.ts` workspace form/server validation.
- `prisma/schema.prisma` data models.

Current Flow Details

Auth flow
1. Register or login request hits Hono auth route.
2. Server creates `Session` row and sets `auth_token` cookie.
3. Middleware reads cookie and resolves user.
4. `/api/auth/me` returns current user when authenticated.
5. Logout deletes session and clears cookie.

Workspace flow
1. Create form validates with Zod.
2. Mutation posts to `/api/workspaces`.
3. Route checks session middleware and writes to `Project`.
4. On success, form resets and query invalidates.

Common Day-1 Tasks

Add a new API route
1. Create feature route file under `features/<feature>/server/route.ts`.
2. Mount route inside `app/api/[...route]/route.ts`.
3. Add zod schema for request validation.
4. Add hook under feature `api` folder.
5. Connect UI form or action.

Add a new form
1. Add zod schema.
2. Build client component with React Hook Form.
3. Submit via mutation hook.
4. Add success and error handling.

Troubleshooting Guide

Issue: login succeeds but user returns to sign-in
1. Check `POST /api/auth/login` returns 200.
2. Check response contains `Set-Cookie` for `auth_token`.
3. Check next `GET /api/auth/me` status.
4. If cookie is missing on localhost, review secure cookie flag in auth route.

Issue: `useForm` export not found / react-server import
1. Ensure form component begins with `"use client"`.
2. Ensure component is not treated as server-only through import chain.

Issue: Event handlers cannot be passed to Client Component props
1. Do not pass raw function props from server component into client component unless boundary is correct.
2. Move interactivity into a client component wrapper.

Issue: redirected to `/dashBoard` 404
1. Use `/` for dashboard root unless route exists.
2. Keep path names consistent with app directory segments.

Known Risks (Current)
- Passwords are currently plain text.
- Cookie policy is not environment-aware in all paths.
- Some frontend code still uses loose error typing (`any`).
- Workspace naming in UI differs from DB model naming (`Project`).

Definition of Done For Local Setup
- `npm run dev` starts without blocking runtime errors.
- Register, login, onboarding, and logout flow works.
- `/api/auth/me` returns 200 when logged in and 401 after logout.
- Workspace creation succeeds from dashboard form.
